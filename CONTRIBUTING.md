# Contributing to FloraLab

Thank you for your interest in contributing. Please note: FloraLab is proprietary software and is distributed under a proprietary "All Rights Reserved" license. Before contributing, read the short notice below about licensing and contributor agreements.

## Short contributor notice

- This repository is covered by a proprietary license. See the `LICENSE` file in the repository root for full terms.
- External contributors who wish to use the code, reproduce it, or include it in other projects must obtain an explicit license from the copyright holder (`trinh-tnp`).
- Contributions submitted to this repository are subject to the project's contribution policy and may require a Contributor License Agreement (CLA) or a separate written license to be signed prior to acceptance.

If you want to contribute code or request a license, please do one of the following:

1. Open an issue titled "License request / Contribution" describing your intended use and contact details (preferred for license requests).
2. Open a pull request with your proposed changes and include a description stating you agree to sign the project CLA if requested. The maintainers will reply with next steps and any required paperwork.

Contact and next steps

- GitHub: https://github.com/trinh-tnp
- When requesting a license or submitting a PR that you wish to have accepted, provide:
  - Your name and contact email
  - The intended use (commercial, internal, research, etc.)
  - Any timelines or distribution requirements

What to include in a PR

- A clear description of the change and why it is needed.
- Small, focused PRs are easier to review.
- Unit tests where applicable.
- Adherence to existing code style and linters.

Review process

- The maintainers will review issues/PRs and respond with any CLA or licensing steps required.
- The maintainers reserve the right to accept, request changes to, or decline contributions.

If you prefer, you can request a commercial license or contributor agreement directly by opening an issue and tagging `@trinh-tnp` in the description.

Thank you for your interest and for respecting the project's licensing terms.
